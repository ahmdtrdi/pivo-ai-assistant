Table 1 — AI confidence tier system (4-gate logic)
Each SKU starts at Green and degrades. Gates are checked in order — first failure sets the tier for that SKU.
Tier Gate 1: Data volume Gate 2: Recency Gate 3: Volatility (CV) Gate 4: Interval width System behaviour
🔴 Red < 7 unique sale dates Last sale > 14 days ago —— No prediction. WA reminder: "Data masih sedikit, terus catat ya!"
🟡 Yellow 7–20 unique sale dates Within 14 days CV > 2.0 (erratic) Interval > 2 × yhat Prediction with caveat: "Prediksi ini masih perkiraan awal."
🟢 Green 21+ unique sale dates Within 14 days CV ≤ 2.0 Interval ≤ 2 × yhat Full forecast + explanation + one concrete action.
Model fallback: if both Prophet MAPE and ARIMA MAPE exceed 40%, SKU is forced to Yellow regardless of data volume.
Table 2 — Forecasting outputs per SKU
All four outputs come from a single Prophet/ARIMA model fit per SKU. See Table 4 for the full list of input features used by the model.
Output field Source Derivation Surfaces in
qty_predicted Prophet / ARIMA forecast['yhat'] — point estimate of units expected to sell tomorrow WA production section
qty_low / qty_high Prophet / ARIMA yhat_lower / yhat_upper — 80% confidence interval around the prediction WA as range: "40–56 cup"
trend_direction Prophet forecast['trend'] delta vs 7 days prior. >+5% = growing, <−5% = declining, else stable LLM tone, PWA chart
stockout_risk Post-model rule qty_predicted > rolling_7d_sales × 1.15 — predicted demand exceeds typical prep by 15% WA stockout alert ⚠️
trend_direction and stockout_risk are derived after the model runs — not direct model outputs. stockout_risk uses rolling_7d_sales from Table 3 as its baseline.
Table 3 — Profit and analysis outputs per SKU
All outputs are deterministic arithmetic computed with pandas. No forecasting model involved. Runs in parallel with the forecasting loop across all active SKUs.
Output field Formula Description Surfaces in
revenue price × qty_sold Total sales value per SKU per day PWA KPI card
gross_profit (price − cost) × qty_sold Actual profit after ingredient cost, per SKU per day WA top products, PWA
margin_pct (price − cost) / price × 100 Percentage of revenue kept as profit — used to rank products WA top products, PWA
rolling_7d_sales avg(qty_sold, 7 days) Smoothed demand baseline — also used in stockout_risk (Table 2) Confidence guard, PWA
rolling_7d_profit avg(gross_profit, 7 days) Weekly profit trend per SKU PWA KPI trend chart
margin_delta margin_pct(week) − margin_pct(prev week) Week-on-week margin change — negative signals cost creep WA margin alert ⚠️
rank_gap volume_rank − profit_rank Gap ≥ 2 means owner is prioritising a high-volume but low-margin SKU PWA dashboard insight
unsold_qty qty_prepared − qty_sold Units prepared but not sold — waste indicator per day PWA waste tracker
sell_through_rate qty_sold / qty_prepared × 100 % of prepared stock sold — low rate = consistent overproduction PWA, WA if critically low
qty_prepared is optional in Google Sheets. If not entered, unsold_qty and sell_through_rate are null for that day without affecting other outputs.
Table 4 — Prophet model features and regressors
Features marked "native" are handled automatically by Prophet. Features marked "regressor" must be added via model.add_regressor(). "Excluded" features cause data leakage or redundancy.
Feature Type Description PoC priority
ds (date) native Prophet's time index. Auto-extracts weekly/yearly seasonality and trend. Required
y (qty_sold) target What the model predicts. One model fit per active SKU. Required
qty_lag_7 regressor Qty sold same day last week. Strongest single predictor for warung demand patterns. Required
qty_lag_14 regressor Qty sold two weeks ago. Confirms whether last week's pattern was a fluke or consistent. Recommended
rolling_mean_7 regressor 7-day moving average of qty_sold. Smooths noise. Also used in stockout_risk. Recommended
is_payday_week regressor 1 if date falls in days 25–2 of month (Indonesian gajian cycle). Demand spikes at month-end. Required
price_changed regressor 1 if unit_price changed vs yesterday. Captures demand sensitivity to price changes. Recommended
is_promo_day regressor Owner-reported discount or event day. Optional column in Google Sheet. Optional
day_of_week built-in Prophet extracts from ds automatically. Do not add as a manual column — double-counts the signal. Do not add
revenue excluded Derived from price × qty — leaks the target variable into the model. Excluded
unit_price (raw) excluded Multicollinear with price_changed. Use price_changed instead. Excluded
qty_lag_0 excluded Today's actual sales — not available at prediction time. Data leakage. Excluded
Start with ds + y + qty_lag_7 + is_payday_week only. Add regressors incrementally as data grows — more features on sparse data degrades accuracy.
Table 5 — Missing data behaviour by consecutive days without input
The system never goes silent. Every nightly run produces either a prediction or a reminder message.
Days without data Prediction behaviour Confidence tier effect WA message sent
0 (data received) Normal pipeline runs No change — tier by 4-gate logic Normal daily insight message
1–2 days Predict using last known data No automatic downgrade yet Insight + gentle reminder: "Jangan lupa catat penjualan hari ini ya!"
3–6 days Continue with stale data Downgrade 1 tier level Reminder first, prediction appended with data-staleness caveat
7+ days No prediction issued Force Red regardless of history Warm reminder only. NGO facilitator also notified.
Data resumes Pipeline resumes next nightly run Tier recalculated via 4-gate logic Normal insight + "Makasih sudah catat lagi!"
Gate 3 (last sale > 14 days → force Red) interacts with this table: an owner who stops for 14+ days hits Red via Gate 3 before the 7-day rule triggers.
