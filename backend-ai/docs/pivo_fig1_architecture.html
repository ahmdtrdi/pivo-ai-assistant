04 · AI / ML — per-SKU loop
for each active SKU (≤ 20 iterations)
Time-series forecasting
Fits a forecasting model on each SKU's full history using time-based regressors. Evaluates model accuracy via cross-validation and selects the best model. (See Table 4 for features, Table 2 for forecasting outputs)
Prophet / NeuralProphet pmdarima (ARIMA fallback) Prophet cross_validation
Margin + profit analysis
Deterministic arithmetic across all SKUs. Computes revenue, gross profit, margin %, rolling 7-day averages, rank gap, unsold qty, and sell-through rate. (See Table 3)
pandas Revenue = price x qty Gross Profit = (price - cost) x qty Margin % = (price - cost) / price x 100 Sell-through Rate = qty_sold / qty_prepared x 100
05 · Confidence guard
Confidence check (per SKU)
Assesses data sufficiency and model reliability for each SKU. Assigns a Red / Yellow / Green tier that controls how the LLM communicates the result. (See Table 1 for full gate logic)
Custom Python logic Prophet confidence intervals Red · Yellow · Green tier output
LLM explanation layer
Receives structured JSON (predictions, tier, flags, rankings). Generates a 3-section Bahasa Indonesia message. Tone adapts to confidence tier: Green gives specific numbers, Yellow adds caveat, Red gives encouragement only.
